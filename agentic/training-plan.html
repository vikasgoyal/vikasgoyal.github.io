Agentic AI Training Plan

A modular curriculum organized into self-contained units. Each module declares its duration, target roles, format, prerequisites, and outcomes, so it can be scheduled into any day/slot.

3. Role → Recommended Modules

Tracks A, B, and G run in sequence (not in parallel). Each attendee joins only the modules listed for their role and is free to leave or join during other modules. This removes the "one room only" constraint, lets a role pick modules from any track, and means the same instructor team can teach all tracks back-to-back.

5. Sample Schedules (mix-and-match)

Modules are independent enough to be re-sequenced. Example schedules:

Option A — Sequential 3-day (Wed/Thu/Fri)

All tracks run back-to-back in one room. Roles attend only the modules relevant to them (per §3) and may step out for the rest.

  • Day 1 — Foundation (2.5 hr, all 200–300): F1 → F2 → F3 → break → F4 → F5 → F6
  • Day 2 — Tracks A → B in sequence (~6 hr with breaks; can split AM/PM):
    • AM block (3.75 hr) — Track A: A1 → A2 → A3 → break → A4 → A5 → A6
      Attending: TA, FA, DL (A6 only), DevL, Dev (A2/A3), DepL (A5)
    • PM block (4.5 hr) — Track B: B1 → B2 → B3 → break → B4 → B5 → B6
      Attending: DevL, Dev, TL (B1/B4/B6), DepL (B5/B6)
  • Day 3 — Track G + Capstone:
    • AM (3 hr) — Track G: G1 → G2 → G3 → break → G4 → G5 → G6
      Attending: DL, PgM, PMO, TA/FA (G3), TL (G2), DepL (G4)
    • PM (3 hr) — Capstone: S1 or S2 (technical roles build; architects/delivery/PMO review)
If 6 hr in one day is too long, split Track A and Track B across Day 2 + Day 3 morning, and push the capstone to a 4th day.

Option B — Spread over a week (lighter daily load)

  • Mon (1 hr): F1 + F2
  • Tue (1.5 hr): F3 + F4 + F5 + F6
  • Wed (2 hr): track-specific morning session
  • Thu (2 hr): track-specific afternoon session
  • Fri (3 hr): capstone scenario

Option C — Two half-days only (executive / leadership audience)

  • Half-day 1 (2.5 hr): F1–F5 + G1
  • Half-day 2 (2.5 hr): G2 + G3 + G4 + G5 + G6 + A6

Option D — Developer bootcamp (technical only, 2 days)

  • Day 1 (4 hr): F1, F2, F3, F6, A2, A3, A4
  • Day 2 (5 hr): B1, B2, B3, B4, B5, B6 + trimmed scenario (S1 or S2, 2 hr)

6. Logistics & Pre-requisites

  • Pre-reads: agentic AI primer, Microsoft Foundry quickstart, LangGraph tour, Microsoft Agent Framework (MAF) tour, MCP spec summary, A2A protocol summary.
  • Lab environment (any B or S module): Azure subscription with Microsoft Foundry project, GitHub repo template, VS Code + recommended extensions.
  • Capacity guidance: F-modules tolerate 200–300 (webinar style); A/B/G modules best at ≤80 per room; S-modules at ≤40 per scenario room with 2 instructors + 2 TAs.
  • Artifacts collected: architecture diagrams (A), runnable repos (B), governance canvases (G), scenario demos (S).
